GDK Demo is a collection of sample programs and example code, and other utility class libraries, for Google Glass Development Kit (GDK). Please check out the project on GitHub: GDK Demo Apps.

Glass Development Kit Playground

GDK Demo is GDK playground for Android/Glassware developers. It includes a variety of sample codes using GDK (or, just plain Android API), which can be quickly built/run to test certain API features or just to get access to some working example apps. All the code samples in the GDKDemo site is released under an open-source license, DNE-1.0.

Building and Running the GDK Sample Apps

Please refer to the GitHub GDK Demo site for dev-related information. The projects include gradle build scripts and should be buildable as is. You can easily import the projects to an IDE such as Android Studio as well.

GDK Live Card Example

Google Glass Development Kit is a "variation" of the standard Android SDK. There are a few new features in GDK that are specific to Google Glass, including the voice trigger, and Glass-specific touch gesture inputs, etc. In my view, the Live Card is one of the most important features in GDK. LiveCard Demo apps on GitHub demonstate basic usages of the LiveCard API.

GDK Touch Gesture Example

Google Glass includes a touch pad, but no touch screen. GDK `GestureDetector` (similar to its Android counterpart) provides a high-level gesture API (as opposed to having to deal with the low-level touch events directly). The Gesture Demo app is based on the GDK doc sample code, and it shows how to handle the Glass gesture events.

GDK Voice Input Example

Google GDK, or the current "Sneak Peek" version, includes two types of supports for voice inputs. One is used to start a Glassware on your Glass. It is called "Voice Trigger", and it can be easily enabled simply changing some configurations or resource files. The second prong of the GDK voice input API is the speech recognition, which is not specific to GDK, but is more valuable on devices like Google Glass. The Voice Demo app demonstrates the use of these two types of voice API.
